# 计划执行 - 任务分解阶段

## 阶段目标
将大任务分解为可执行的小任务。

## 分解原则

### SMART 原则
- **S**pecific：具体明确
- **M**easurable：可衡量
- **A**chievable：可实现
- **R**elevant：相关性
- **T**ime-bound：有时限

### 分解方法
```
大任务
├── 子任务 1
│   ├── 步骤 1.1
│   └── 步骤 1.2
├── 子任务 2
│   ├── 步骤 2.1
│   └── 步骤 2.2
└── 子任务 3
```

## 任务模板

```markdown
## 任务：实现用户登录功能

### 子任务
1. [ ] 设计数据库表结构
2. [ ] 实现后端 API
3. [ ] 实现前端页面
4. [ ] 编写测试用例
5. [ ] 集成测试

### 依赖关系
- 任务 2 依赖任务 1
- 任务 3 依赖任务 2
- 任务 4 可并行
```

## 依赖分析

### 依赖类型
| 类型 | 说明 |
|------|------|
| 强依赖 | 必须先完成 |
| 弱依赖 | 最好先完成 |
| 无依赖 | 可并行执行 |

## 分解清单

- [ ] 任务是否足够小
- [ ] 依赖关系是否清晰
- [ ] 是否可独立执行
