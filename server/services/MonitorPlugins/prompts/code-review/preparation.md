# 代码审查 - 准备阶段

## 阶段目标
了解代码变更的背景和范围，为审查做好准备。

## 审查前准备

### 1. 了解变更背景
- 阅读 PR/MR 描述
- 了解相关的 Issue 或需求
- 理解变更的目的和预期效果

### 2. 确定审查范围
```bash
# 查看变更的文件列表
git diff --name-only main...feature-branch

# 查看变更统计
git diff --stat main...feature-branch

# 查看具体变更
git diff main...feature-branch
```

### 3. 审查清单准备

| 审查维度 | 关注点 |
|---------|-------|
| 功能正确性 | 是否实现了预期功能 |
| 代码质量 | 命名、结构、可读性 |
| 安全性 | 输入验证、权限检查 |
| 性能 | 算法效率、资源使用 |
| 测试覆盖 | 是否有足够的测试 |
| 文档 | 注释、API 文档 |

## 审查优先级

### 高优先级
- 安全相关代码
- 核心业务逻辑
- 公共 API 变更
- 数据库操作

### 中优先级
- 新功能实现
- 重构代码
- 配置变更

### 低优先级
- 格式调整
- 注释更新
- 文档修改

## 检查清单

- [ ] 已阅读 PR 描述和相关 Issue
- [ ] 已了解变更的目的
- [ ] 已确定审查范围
- [ ] 已准备审查清单
