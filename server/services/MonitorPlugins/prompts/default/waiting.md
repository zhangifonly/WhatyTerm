# 通用监控 - 等待输入状态

## 状态说明
程序正在等待用户输入。

## 常见等待类型

### 1. 继续确认
Claude Code 等待"继续"指令

**特征**:
- 显示 `>` 提示符
- 之前有任务输出
- 没有错误信息

**操作**: 发送 `继续`

### 2. 是/否确认
需要 y/n 响应

**特征**:
- 提示 `(y/n)`, `[Y/n]`, `[yes/no]`
- 询问是否执行某操作

**操作**: 根据上下文选择
- 安装依赖 → `y`
- 删除文件 → 谨慎确认后 `y`
- 覆盖文件 → 确认备份后 `y`
- 不确定 → `n`（保守选择）

### 3. 选项选择
需要选择数字

**特征**:
- 显示 `[1]`, `[2]`, `[3]` 等选项
- 每个选项有说明文字

**操作**: 阅读选项说明后选择

**Claude Code 权限确认**:
```
[1] Allow once (本次允许)
[2] Allow for this session (本次会话允许)
[3] Deny (拒绝)
```
**推荐**: 选择 `2`，避免频繁确认

### 4. 文本输入
需要输入具体内容

**特征**:
- 提示输入文件名、路径、配置等
- 显示输入框或冒号提示

**操作**: 根据提示输入相应内容

## 安全检查

在自动响应前，确认：

- [ ] 操作不会造成数据丢失
- [ ] 操作不会影响生产环境
- [ ] 操作是可逆的或有备份
- [ ] 理解操作的影响范围

## 不确定时的处理

如果不确定应该输入什么：
1. 选择保守选项（n、取消、跳过）
2. 等待人工确认
3. 查看帮助信息（如果有）
