# 通用监控 - 运行中状态

## 状态说明
程序正在执行中，请耐心等待。

## 识别特征

### 进度指示
- 百分比显示：`45%`, `100/200`
- 进度条：`[=====>    ]`
- Spinner 动画：`⠋`, `⠙`, `⠹`, `⠸`
- ETA 显示：`ETA: 2m30s`

### 持续输出
- 日志信息滚动
- 编译信息输出
- 下载进度更新
- 测试用例执行

### 状态提示词
- `running`, `processing`, `loading`
- `building`, `compiling`, `installing`
- `downloading`, `uploading`

## 监控要点

### 1. 进度是否正常推进
- 观察进度数字是否在增加
- 观察输出是否在更新
- 如果长时间（>5分钟）无变化，可能卡住

### 2. 是否有警告信息
- `warning` 可能预示后续错误
- `deprecated` 提示使用了过时的 API
- 记录警告信息，任务完成后处理

### 3. 资源使用情况
- CPU 使用率是否异常高
- 内存是否持续增长
- 磁盘空间是否充足

## 异常处理

### 程序卡住
**症状**: 超过 5 分钟无进度变化

**处理**:
1. 检查是否在等待用户输入
2. 检查网络连接
3. 考虑中断（Ctrl+C）重试

### 超时警告
**症状**: 出现 `timeout`, `timed out` 提示

**处理**:
1. 检查网络连接
2. 增加超时时间配置
3. 重试操作

### 内存警告
**症状**: 出现 `out of memory`, `heap` 相关错误

**处理**:
1. 增加 Node.js 内存限制：`--max-old-space-size=4096`
2. 优化代码减少内存使用
3. 分批处理大数据

## 操作建议

**当前状态**: 运行中，无需操作

等待程序执行完成，不要发送任何指令打断执行。
