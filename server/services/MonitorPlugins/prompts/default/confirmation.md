# 通用监控 - 确认界面

## 状态说明
检测到需要用户确认的界面。

## Claude Code 权限确认

当看到类似以下选项时：
```
[1] Allow once (本次允许)
[2] Allow for this session (本次会话允许)
[3] Deny (拒绝)
```

**推荐选择**: `2` - 允许本次会话

**原因**:
- 避免频繁确认打断工作流
- 会话结束后权限自动失效
- 比"始终允许"更安全

## 其他确认类型

### y/n 确认

根据操作内容判断：

| 操作类型 | 建议 | 说明 |
|---------|------|------|
| 安装依赖 | `y` | 通常安全 |
| 更新依赖 | `y` | 注意版本兼容 |
| 删除文件 | 谨慎 | 确认是否有备份 |
| 覆盖文件 | 谨慎 | 确认是否需要保留 |
| 初始化项目 | `y` | 确认目录正确 |
| 发布/部署 | 谨慎 | 确认环境正确 |

### 数字选择

1. 阅读每个选项的说明
2. 理解每个选项的影响
3. 选择最合适的选项

### 文本输入

1. 理解提示要求什么
2. 输入正确格式的内容
3. 不确定时查看帮助或示例

## 安全原则

### 谨慎操作
- 涉及删除、覆盖时先确认备份
- 涉及生产环境时需要人工确认
- 涉及敏感信息时检查安全性

### 保守选择
- 不确定时选择"否"或"取消"
- 可以稍后重新执行
- 避免不可逆的操作

### 人工确认场景
- 删除大量文件
- 修改系统配置
- 部署到生产环境
- 涉及付费操作
