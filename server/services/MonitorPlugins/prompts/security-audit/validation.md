# 安全审计 - 漏洞验证阶段

## 阶段目标
验证发现的漏洞是否可利用。

## 验证原则

### 安全第一
- 获得授权
- 不造成破坏
- 记录所有操作
- 保护敏感数据

### 最小影响
- 使用最小权限
- 避免 DoS
- 不修改数据
- 及时清理痕迹

## 验证方法

### SQL 注入验证
```sql
-- 基本测试
' OR '1'='1
' OR '1'='1' --
' UNION SELECT NULL --

-- 时间盲注
' AND SLEEP(5) --
```

### XSS 验证
```html
<script>alert('XSS')</script>
<img src=x onerror=alert('XSS')>
```

### 命令注入验证
```bash
; id
| whoami
`id`
$(id)
```

## 证据收集

### 截图
- 漏洞触发前
- 漏洞触发时
- 漏洞影响

### 日志
- 请求内容
- 响应内容
- 时间戳
