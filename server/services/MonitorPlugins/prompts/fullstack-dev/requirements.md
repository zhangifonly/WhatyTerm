# 全栈开发 - 需求文档阶段

## 阶段目标
制定完整、清晰、可执行的需求文档，为后续开发提供明确指导。

## 需求编写原则

### SMART 原则
- **S**pecific（具体）：需求描述清晰，无歧义
- **M**easurable（可衡量）：有明确的验收标准
- **A**chievable（可实现）：技术上可行
- **R**elevant（相关）：与项目目标一致
- **T**ime-bound（有时限）：有明确的优先级

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| 需求过于笼统 | 细化为可测试的功能点 |
| 缺少边界情况 | 考虑异常流程和错误处理 |
| 验收标准模糊 | 使用可量化的指标 |
| 优先级不清 | 使用 P0/P1/P2 分级 |
