# 全栈开发 - 测试归档阶段

## 阶段目标
全面测试项目功能，发现并修复 bug，确保项目质量达标。

## 测试流程

### 1. 制定测试计划
- 确定测试范围
- 编写测试用例
- 准备测试数据

### 2. 执行测试
- 按用例逐项测试
- 记录测试结果
- 发现 bug 立即记录

### 3. Bug 修复
- 分析 bug 原因
- 修复 bug
- 回归测试确认

### 4. 测试报告
- 汇总测试结果
- 记录遗留问题
- 归档测试文档

## 边界测试清单

| 测试场景 | 输入 | 预期结果 |
|---------|------|---------|
| 空值输入 | 空字符串 | 显示必填提示 |
| 超长输入 | 1000+ 字符 | 截断或提示 |
| 特殊字符 | `<script>alert(1)</script>` | 正确转义 |
| SQL 注入 | `'; DROP TABLE users; --` | 无影响 |
| 负数输入 | -1 | 显示错误或处理 |
| 边界值 | 0, MAX_INT | 正确处理 |

## Bug 处理原则

### 不跳过任何 bug
所有发现的 bug 必须修复，不能绕过或忽略。
