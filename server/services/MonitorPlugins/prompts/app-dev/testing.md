# App 开发 - 测试调试阶段

## 阶段目标
发现并修复问题，确保功能正确。

{{include:common/testing}}

## 调试技巧

### 错误分析
1. 阅读错误信息
2. 定位错误位置
3. 分析错误原因
4. 制定修复方案

### 常用调试方法
```javascript
// 断点调试
debugger;

// 日志调试
console.log('变量值:', variable);
console.trace('调用栈');

// 条件断点
// 在 DevTools 中设置
```

### 错误类型
| 类型 | 示例 | 处理方式 |
|------|------|----------|
| 语法错误 | SyntaxError | 检查代码语法 |
| 类型错误 | TypeError | 检查变量类型 |
| 引用错误 | ReferenceError | 检查变量定义 |
| 逻辑错误 | 结果不符预期 | 检查业务逻辑 |
